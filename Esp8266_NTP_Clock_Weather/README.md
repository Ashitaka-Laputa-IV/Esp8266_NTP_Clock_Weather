# ESP8266 NTP时钟天气站

基于ESP8266的智能时钟天气显示系统，集成和风天气API，支持NTP时间同步、OLED显示和温度监测。

## 🚀 项目特色

- **智能时钟**：通过NTP协议同步网络时间，精准可靠
- **实时天气**：集成和风天气API，显示当前天气状况
- **天气预报**：显示未来3天天气趋势
- **温度监测**：使用DS18B20传感器监测环境温度
- **智能配网**：支持WiFi自动连接和网页配网模式
- **美观界面**：OLED显示屏，支持多页面切换动画

## 📦 硬件要求

### 必备硬件
- **ESP8266开发板**：NodeMCU、Wemos D1 Mini等
- **OLED显示屏**：SSD1306或SH1106芯片，128×64分辨率
- **DS18B20温度传感器**：监测环境温度
- **WiFi网络**：2.4GHz WiFi网络

### 可选硬件
- 面包板和杜邦线（用于连接）
- 3D打印外壳（可选）
- 电源适配器（5V/2A）

## 🔌 硬件连接

### 引脚连接表
| 组件 | ESP8266引脚 | 说明 |
|------|------------|------|
| OLED SDA | D2 (GPIO4) | I2C数据线 |
| OLED SCL | D5 (GPIO14) | I2C时钟线 |
| DS18B20数据线 | D7 (GPIO13) | 温度传感器数据线 |
| DS18B20 VCC | 3.3V | 电源正极 |
| DS18B20 GND | GND | 电源地线 |

### 连接示意图
```
ESP8266
  │
  ├── OLED显示屏
  │   ├── SDA → D2
  │   ├── SCL → D5
  │   ├── VCC → 3.3V
  │   └── GND → GND
  │
  └── DS18B20传感器
      ├── 数据线 → D7
      ├── VCC → 3.3V
      └── GND → GND
```

## ⚙️ 软件配置

### 1. 安装Arduino IDE
1. 下载并安装 [Arduino IDE](https://www.arduino.cc/en/software)
2. 安装ESP8266开发板支持
   - 文件 → 首选项 → 附加开发板管理器网址：`http://arduino.esp8266.com/stable/package_esp8266com_index.json`
   - 工具 → 开发板 → 开发板管理器 → 搜索"esp8266" → 安装

### 2. 安装必要的库
在Arduino IDE中安装以下库：
- **ESP8266WiFi**（内置）
- **ESP8266WebServer**（内置）
- **DS18B20** by Matthias Busse
- **SH1106Wire** by ThingPulse
- **OLEDDisplayUi** by ThingPulse

安装方法：工具 → 管理库 → 搜索库名 → 安装

### 3. 配置项目参数
打开 `main_simple.ino` 文件，修改以下配置：

```cpp
// WiFi网络配置
const char* WIFI_SSID = "您的WiFi名称";
const char* WIFI_PASSWORD = "您的WiFi密码";

// 和风天气API配置
const char* HEFENG_KEY = "您的API密钥";
const char* HEFENG_LOCATION = "城市代码"; // 如："beijing"
```

### 4. 获取和风天气API密钥
1. 访问 [和风天气开发者平台](https://dev.heweather.com/)
2. 注册账号并登录
3. 创建应用，获取API密钥
4. 选择免费版（每天1000次调用）

## 🛠️ 使用指南

### 快速开始

1. **硬件连接**：按照引脚连接表连接所有硬件
2. **软件配置**：修改配置文件中的WiFi和API密钥
3. **编译上传**：选择正确的开发板和端口，点击上传
4. **启动设备**：设备启动后会自动连接WiFi

### 首次使用流程

1. **设备启动**：上电后设备开始初始化
2. **WiFi连接**：
   - 如果已保存WiFi凭据：自动连接
   - 如果连接失败：启动网页配网模式
3. **时间同步**：连接NTP服务器获取准确时间
4. **天气获取**：首次获取天气数据
5. **正常显示**：开始循环显示时间、天气、温度

### 网页配网模式
如果自动连接失败，设备会启动AP模式：
- **热点名称**：`weather_clock`
- **访问地址**：`http://192.168.4.1`
- **操作步骤**：
  1. 手机/电脑连接 `weather_clock` 热点
  2. 浏览器访问 `http://192.168.4.1`
  3. 填写WiFi名称和密码
  4. 点击连接，等待设备重启

## 📱 功能说明

### 显示页面
设备会循环显示3个页面：

1. **日期时间页面**
   - 显示当前日期（年-月-日 星期）
   - 显示当前时间（时:分:秒）
   - 底部显示当前时间和传感器温度

2. **当前天气页面**
   - 显示当前温度（大字体）
   - 显示体感温度和湿度
   - 显示天气状况和风力
   - 显示对应天气图标

3. **天气预报页面**
   - 显示未来3天天气预报
   - 每项显示日期、天气图标、温度范围

### 数据更新
- **时间更新**：实时更新，每秒刷新
- **温度更新**：每10秒更新一次
- **天气更新**：每20分钟更新一次
- **网络重连**：连接失败时自动重试

## 🔧 故障排除

### 常见问题

#### 1. 无法连接WiFi
- **检查**：WiFi名称和密码是否正确
- **解决**：使用网页配网模式重新配置

#### 2. 时间显示不正确
- **检查**：时区设置是否正确（中国为UTC+8）
- **解决**：修改 `TIMEZONE` 参数为 `-8`

#### 3. 天气数据获取失败
- **检查**：API密钥和城市代码是否正确
- **解决**：检查和风天气API配额是否用完

#### 4. OLED显示屏不亮
- **检查**：电源和引脚连接是否正确
- **解决**：确认I2C地址是否为 `0x3c`

#### 5. 温度显示异常
- **检查**：DS18B20连接是否正确
- **解决**：确认数据线连接D7引脚

### 调试技巧

1. **启用串口监视器**：波特率115200，查看调试信息
2. **检查网络状态**：观察WiFi连接和API响应
3. **验证硬件连接**：使用万用表检查引脚连接
4. **更新库文件**：确保所有库都是最新版本
